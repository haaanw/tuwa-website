The FeatureGrid has two translation challenges:
1. Static HTML (heading, SVG text labels, initial center content, sr-only button text)
2. Inline script `features` array used for dynamic center content on wheel rotation

**Strategy:** Use Astro frontmatter to get translations, render static HTML with template expressions, and pass the features data to the inline script via a `<script>` tag that sets a global variable (since `<script is:inline>` cannot use `define:vars`).

**1. Add frontmatter:**

At the top of FeatureGrid.astro, add a frontmatter block:
```astro
---
import { useHomeTranslations } from '../i18n/utils';
import type { Locale } from '../i18n/utils';
import { getRelativeLocaleUrl } from 'astro:i18n';

interface Props {
  locale?: Locale;
}

const { locale = 'en' } = Astro.props;
const t = useHomeTranslations(locale);

// Build features array with locale-aware hrefs for the inline script
const featuresData = t.featureGrid.features.map((f, i) => ({
  title: f.title,
  desc: f.desc,
  href: getRelativeLocaleUrl(locale, f.href),
}));
---
```

**2. Replace static HTML text:**

- The `<h2>` heading: replace `Everything you need to train without guessing` with `{t.featureGrid.heading}`

- SVG segment labels (5 `<text>` elements with class `wheel-segment-label`):
  - `RECOVERY` -> `{t.featureGrid.segmentLabels[0]}`
  - `LOAD` -> `{t.featureGrid.segmentLabels[1]}`
  - `TRAINING` -> `{t.featureGrid.segmentLabels[2]}`
  - `ONBOARDING` -> `{t.featureGrid.segmentLabels[3]}`
  - `COACHING` -> `{t.featureGrid.segmentLabels[4]}`

- Button sr-only text (5 `<span class="sr-only">` elements):
  - `Recovery Scoring` -> `{t.featureGrid.features[0].title}`
  - `Workload Tracking` -> `{t.featureGrid.features[1].title}`
  - `Smart Templates` -> `{t.featureGrid.features[2].title}`
  - `Cold-Start Onboarding` -> `{t.featureGrid.features[3].title}`
  - `Coach + Athlete` -> `{t.featureGrid.features[4].title}`

- Center overlay initial content:
  - `<span class="wheel-center-title">Recovery Scoring</span>` -> `<span class="wheel-center-title">{t.featureGrid.features[0].title}</span>`

**3. Pass translated features data to inline script:**

BEFORE the existing `<script is:inline>` tag, add a data-passing script:
```astro
<script is:inline define:vars={{ featuresData, exploreCta: t.featureGrid.exploreCta }}>
  window.__featureGridData = { features: featuresData, exploreCta: exploreCta };
</script>
```

Note: `define:vars` works with `is:inline` scripts in Astro. It serializes the variables as JSON inline.
