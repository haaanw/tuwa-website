## Interaction Contract

Source: CONTEXT.md decisions D-04 through D-06. These are binding.

### Button / CTA micro-interactions (D-04)

| State | Transform | Duration | Easing |
|-------|-----------|----------|--------|
| Default | `scale(1)` | — | — |
| Hover | `scale(1.02)` | 150ms | `ease-out` |
| Active (`:active`) | `scale(0.98)` | 100ms | `ease-out` |

Apply to: all `<a>` and `<button>` elements styled as CTAs — Hero download button, LandingCTA App Store badge, FeatureCTA buttons, Header nav CTA.

CSS pattern (add to `global.css`):

```css
.btn-cta {
  transition: transform 150ms ease-out;
}
.btn-cta:hover {
  transform: scale(1.02);
}
.btn-cta:active {
  transform: scale(0.98);
}
```

### Card hover lift (D-05)

| State | Shadow | Duration | Easing |
|-------|--------|----------|--------|
| Default | `var(--shadow-card)` | — | — |
| Hover | `var(--shadow-card-hover)` | 200ms | `ease-out` |

Apply to: all `.feature-card` elements in FeatureGrid, blog listing cards.

No `translateY` — shadow depth only (D-05 explicit: "no translate").

### Nav link hover states (D-06)

| State | Color | Duration | Easing |
|-------|-------|----------|--------|
| Default | `var(--color-text-2)` | — | — |
| Hover | `var(--color-text-1)` | 150ms | `ease-out` |

Apply to: Header nav links, Footer links, MobileMenu links.

### Prefers-reduced-motion

All hover/active transitions must be wrapped in `@media (prefers-reduced-motion: no-preference)`. At reduced-motion, interactive states change instantly (no transition property).

## Noise Texture Contract (UIPX-04, D-01)

CSS-only, SVG filter approach — no image asset.

**Application:** Full page background pseudo-element. Applied as `body::after` (or `body::before` if `::after` conflicts with existing patterns — check during implementation).

**Parameters (Claude's Discretion):**

| Property | Value | Rationale |
|----------|-------|-----------|
| SVG filter type | `feTurbulence` + `feColorMatrix` | Standard CSS noise technique, zero network cost |
| `baseFrequency` | `0.65` | Medium grain — visible at close inspection, not distracting |
| `numOctaves` | `4` | Enough detail for organic texture without harshness |
| Opacity | `0.025` (2.5%) | Very subtle — references Whoop/Oura grain level (D-01 specifics) |
| Blend mode | `multiply` | Integrates with travertine warm tones; darkens imperceptibly |
| `pointer-events` | `none` | Must not interfere with interactive elements |
| `z-index` | `0` with page content at `z-index: 1` | Texture sits behind all content |

**SVG filter string:**
```css
body::after {
  content: '';
  position: fixed;
  inset: 0;
  pointer-events: none;
  z-index: 0;
  opacity: 0.025;
  background-image: url("data:image/svg+xml,%3Csvg viewBox='0 0 256 256' xmlns='http://www.w3.org/2000/svg'%3E%3Cfilter id='noise'%3E%3CfeTurbulence type='fractalNoise' baseFrequency='0.65' numOctaves='4' stitchTiles='stitch'/%3E%3C/filter%3E%3Crect width='100%25' height='100%25' filter='url(%23noise)'/%3E%3C/svg%3E");
  background-repeat: repeat;
  background-size: 256px 256px;
}
```

Note: Use `position: fixed` not `position: absolute` so the texture tiles continuously during scroll without repositioning.

## Bento Grid Contract (UIPX-05, D-07 through D-09)

### Hierarchy (D-08)

| Tier | Card | Visual Weight |
|------|------|---------------|
| Large (hero) | Recovery Scoring | ~2x column width, taller min-height |
| Medium | Workload Tracking, Smart Templates | Standard 1-column |
| Small | Cold Start Onboarding, Coaching | Compact, condensed content density |

### CSS Grid Template (Claude's Discretion, D-09)

**Desktop (≥ 768px):** 3-column grid, 3-row layout

```
[ Recovery Scoring (col 1-2, row 1-2) ][ Workload Tracking (col 3, row 1) ]
[ Recovery Scoring (continued)        ][ Smart Templates   (col 3, row 2) ]
[ Cold Start (col 1-2) ][ Coaching (col 3) ]  — OR —
[ Cold Start (col 1) ][ Coaching (col 2-3) ]
```

Recommended grid-template-areas:
```css
.bento-grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: auto auto auto;
  gap: var(--space-xl);
  grid-template-areas:
    "recovery  recovery  workload"
    "recovery  recovery  templates"
    "cold-start cold-start coaching";
}
```

**Tablet (480px–767px):** 2-column grid

```css
.bento-grid {
  grid-template-columns: 1fr 1fr;
  grid-template-areas:
    "recovery  recovery"
    "workload  templates"
    "cold-start coaching";
}
```

**Mobile (< 480px):** Single column, natural stacking order: Recovery → Workload → Templates → Cold Start → Coaching.

## Animated Counter Contract (UIPX-06, D-10 through D-12)

### Metrics (Claude's Discretion, D-10)

| Metric | Display Value | Label |
|--------|--------------|-------|
| Athletes tracked | 1,200+ | Athletes using Tuwa |
| Training sessions | 85,000+ | Sessions analyzed |
| Recovery accuracy | 94% | HRV prediction accuracy |

**Rationale:** Three metrics gives visual balance (odd number). Session count and athlete count are social proof. Accuracy percentage adds scientific credibility — core brand value.

### Placement (Claude's Discretion, D-11)

**Between FeatureGrid and LandingCTA on `index.astro`.** This is the conversion-critical position: users who scrolled through all features are primed for social proof before the final CTA. Placement after hero (between hero and features) risks overwhelming a user who hasn't yet understood what the product does.

### Layout

| Property | Value |
|----------|-------|
| Background | `var(--color-surface)` — visual break from bg, consistent with CTA section |
| Border | `border-top: 1px solid var(--color-divider); border-bottom: 1px solid var(--color-divider)` |
| Internal padding | `var(--space-2xl)` (48px) top/bottom |
| Grid | 3-column flex row (`display: flex; justify-content: center; gap: var(--space-3xl)`) |
| Mobile | Stack vertically with `var(--space-xl)` (32px) gap |

### Counter number typography

```
font-size: var(--text-display);  /* 48px */
font-weight: 700;
line-height: var(--leading-display);  /* 1.1 */
font-variant-numeric: tabular-nums;  /* .tabular-nums class */
color: var(--color-text-1);
```

### Counter label typography

```
font-size: var(--text-label);  /* 13px */
font-weight: 400;
letter-spacing: var(--tracking-label);  /* 0.06em */
text-transform: uppercase;
color: var(--color-text-2);
margin-top: var(--space-xs);  /* 4px */
```

### Animation (D-12)

| Property | Value |
|----------|-------|
| Trigger | `data-animate` on counter container — hooks into existing `AnimationController` IntersectionObserver |
| Duration | 400ms — within D-12 300–500ms range |
| Easing | Linear (number increments feel mechanical and deliberate) |
| Start value | 0 (or 0%) |
| End value | Target number |
| Update interval | `requestAnimationFrame` — smooth, no `setInterval` jank |
| Reduced motion | Show final number immediately when `prefers-reduced-motion: reduce` |

**Counter JS hook pattern** (extends AnimationController in BaseLayout.astro):

```js
// On .is-visible, read data-counter-target and data-counter-suffix
// Animate from 0 to target over 400ms using rAF
// Respect window.matchMedia('(prefers-reduced-motion: reduce)')
```

Counter component attributes:
- `data-counter-target="1200"` — numeric end value
- `data-counter-suffix="+"` — appended suffix ("+", "%")
- `data-counter-prefix=""` — optional prefix (none for these metrics)
