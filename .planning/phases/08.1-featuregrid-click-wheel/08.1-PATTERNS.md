#### is:inline IIFE script wrapper pattern

Every browser-side script in this codebase is wrapped in an IIFE `(function() { ... })()` when using `is:inline`. This prevents variable leakage to global scope. The Header uses `is:inline`; MobileMenu uses a plain `<script>` (Astro-compiled). The wheel should use `is:inline` to match BaseLayout's AnimationController pattern (same page, no module bundling concern).

**Analog: `src/components/Header.astro` lines 117–139:**
```html
<script is:inline>
  (function() {
    var dropdown = document.getElementById('features-dropdown');
    if (!dropdown) return;
    var btn = dropdown.querySelector('button');
    var menu = dropdown.querySelector('ul');
    var chevron = btn.querySelector('svg');

    btn.addEventListener('click', function() {
      var open = menu.classList.toggle('hidden');
      btn.setAttribute('aria-expanded', !open);
      chevron.style.transform = open ? '' : 'rotate(180deg)';
    });

    document.addEventListener('click', function(e) {
      if (!dropdown.contains(e.target)) {
        menu.classList.add('hidden');
        btn.setAttribute('aria-expanded', 'false');
        chevron.style.transform = '';
      }
    });
  })();
</script>
```

**Key conventions to copy:**
- `is:inline` attribute on `<script>` tag
- IIFE wrapper `(function() { ... })()`
- `var` declarations (not `let`/`const`) — consistent with BaseLayout AnimationController
- `document.getElementById` / `querySelector` for element selection
- Guard clause `if (!el) return;` at top of IIFE
- Event listeners added via `addEventListener`, never inline `on*` attributes inside the script

#### requestAnimationFrame pattern

**Analog: `src/layouts/BaseLayout.astro` lines 62–76:**
```javascript
var duration = 400;
var start = null;
function step(timestamp) {
  if (!start) start = timestamp;
  var progress = Math.min((timestamp - start) / duration, 1);
  counter.textContent = Math.floor(progress * target).toLocaleString() + suffix;
  if (progress < 1) {
    requestAnimationFrame(step);
  } else {
    counter.textContent = target.toLocaleString() + suffix;
  }
}
requestAnimationFrame(step);
```

**Copy this pattern for the inertia rAF loop:**
- Named function reference (not arrow function) — `function inertiaFrame() { ... }`
- Call `requestAnimationFrame(inertiaFrame)` recursively while condition holds
- Terminal case exits without scheduling another frame (instead of `else cancelAnimationFrame`)
- Store rAF ID in a variable (`var rafId`) so it can be cancelled on `pointerdown` during active inertia

## Critical Implementation Notes

### Do NOT remove `.feature-card` CSS class
Grep confirms `.feature-card` is used in `src/components/FeatureGrid.astro` and `src/styles/global.css` only — but the feature deep-dive page layouts (`FeaturePageLayout.astro`) may inherit it. Per RESEARCH.md Pitfall 5: only remove `.bento-grid` and `.bento-hero-card`. Verify with `grep -r "feature-card" src/` before deleting.

### Shadow tokens to remove with bento classes
`--shadow-bento-hero` and `--shadow-bento-hero-hover` tokens (global.css lines 69–70) are only used by `.bento-hero-card`. They can be removed when `.bento-hero-card` is removed. Verify with grep first.

### SVG transform-origin — use attribute approach
Per RESEARCH.md Pattern 4 and Pitfall 2: do NOT rely on CSS `transform` property on the rotating `<g>` element for drag (cross-browser `transform-origin` bugs). Use `groupEl.setAttribute('transform', 'rotate(N, 240, 240)')` for drag, switch to CSS transition only for the final snap step (or use Web Animations API `el.animate([...], { duration: 200, easing: 'ease-out' })`).

### Button positioning — use percentage-based values
Per RESEARCH.md Assumption A3: compute button `left`/`top` as `(x / 480 * 100)%` and `(y / 480 * 100)%` from `polarToCartesian` results. Do not hardcode pixel values — SVG scales via CSS `width` but viewBox stays 0 0 480 480, so percentages self-correct at all breakpoints.
