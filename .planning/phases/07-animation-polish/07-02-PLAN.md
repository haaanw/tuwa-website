**Add StickyScrollController** — Append a `<script is:inline>` block at the very end of the file, AFTER the closing `</FeaturePageLayout>` tag:

```astro
<script is:inline>
  /* StickyScrollController — scoped to recovery-scoring.astro only */
  /* Separate observer from global AnimationController (different threshold, continuous observation) */
  (function () {
    var steps = document.querySelectorAll('.scroll-step');
    var dots = document.querySelectorAll('.step-dot');
    if (!steps.length) return;

    var observer = new IntersectionObserver(function (entries) {
      entries.forEach(function (entry) {
        if (entry.isIntersecting) {
          var stepNum = entry.target.getAttribute('data-step');
          /* Update active step */
          steps.forEach(function (s) { s.classList.remove('is-active'); });
          entry.target.classList.add('is-active');
          /* Update indicator dots */
          dots.forEach(function (d, i) {
            d.classList.toggle('is-active', String(i + 1) === stepNum);
          });
        }
      });
    }, { threshold: 0.5 });

    steps.forEach(function (el) { observer.observe(el); });
  })();
</script>
```

Key implementation details:
- ES5 `var` and `function` — no const/let/arrows (per 05-PATTERNS.md convention).
- Threshold 0.5 (not 0.15 like global AnimationController) — step activates when 50% visible.
- Does NOT call `observer.unobserve()` — must observe continuously as user scrolls back up.
- `.scroll-step` elements do NOT have `data-animate` — exclusively managed by StickyScrollController.
- Single `recovery.png` for all steps — no image swap needed; dot indicators carry step progression.
- Step dots have `aria-hidden="true"` — purely decorative.
