## Spacing Scale

Declared values (multiples of 4). All tokens live in `src/styles/global.css :root`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-xs`) | Icon gaps, inline padding |
| sm | 8px (`--space-sm`) | Compact element spacing |
| md | 16px (`--space-md`) | Default element spacing, horizontal page padding |
| lg | 24px (`--space-lg`) | Section padding, heading margins |
| xl | 32px (`--space-xl`) | Layout gaps |
| 2xl | 48px (`--space-2xl`) | Hero device margin, major section breaks |
| 3xl | 64px (`--space-3xl`) | Page-level top/bottom section padding |

**Phase 6 exceptions:**
- DeviceFrame component inner padding: 12px top, 8px sides, 8px bottom — these are frame inset values, not spacing between page elements. They do not need to fall on the 8-point scale because they govern hardware chrome reproduction.
- Dynamic Island cutout height: 12px. Home indicator bar height: 5px. These are hardware dimension approximations.

**Source:** `src/styles/global.css` `:root` spacing tokens — pre-populated, not derived from user input.

**Accent reserved for:** CTA buttons only (`Download on the App Store`, `Get Tuwa Free`). Never applied to links, borders, icons, or device frame chrome.

**DeviceFrame color contract (Phase 6 specific):**
- Frame body: near-black, specifically `#1A1A1A` — dark enough to read as iPhone Space Black against the `--color-bg` travertine background. This is a new value not in the token set; apply as a hardcoded CSS value in `DeviceFrame.astro` with an inline comment referencing D-02 from CONTEXT.md.
- Frame highlight (top edge subtle sheen): `rgba(255,255,255,0.08)` — 1px inset border-top on the frame body to suggest metal edge reflection.
- Dynamic Island: `#0D0D0D` — slightly darker than frame body.
- Side buttons (pseudo-elements): same `#1A1A1A` as frame body with `#2A2A2A` right-face to imply depth.
- Home indicator bar: `rgba(255,255,255,0.3)` pill on dark frame base.
- Drop shadow on frame: `0 24px 64px rgba(28,25,21,0.22), 0 4px 16px rgba(28,25,21,0.12)` — premium depth without overwhelming the travertine background.

**Source:** `src/styles/global.css` for all tokens; frame-specific colors decided from D-02 CONTEXT.md (dark frame against travertine) and design discretion.

## Component Contract: DeviceFrame.astro

This is the primary deliverable of Phase 6. Prescribe exact dimensions to eliminate ambiguity.

### iPhone 15 Pro Reference Dimensions

All dimensions are CSS pixels (display units). At 3x DPR the rendered image is crisp.

| Dimension | Value | Notes |
|-----------|-------|-------|
| Frame outer width | 280px (mobile) → 300px (md) → 320px (lg+) | Fluid, not fixed |
| Frame aspect ratio | 9 / 19.5 | iPhone 15 Pro portrait — matches existing `ScreenshotBlock` placeholder |
| Corner radius (outer) | 44px | Matches existing Hero `rounded-[44px]` — keeps visual continuity |
| Corner radius (inner screen) | 36px | 8px tighter than outer to give frame thickness feel |
| Frame border width | 10px | Represents device body thickness on all sides |
| Dynamic Island width | 120px | Centered, pill shape |
| Dynamic Island height | 34px | Positioned 12px from inner screen top edge |
| Dynamic Island corner radius | 20px | Full pill |
| Side button width | 4px | Volume/power pseudo-elements |
| Volume button height | 28px | Each volume button (2 buttons, 6px gap between) |
| Power button height | 42px | Right side |
| Home indicator width | 120px | Bottom center pill |
| Home indicator height | 5px | Standard iOS indicator |
| Home indicator offset | 8px | From inner screen bottom edge |

### Responsive Sizing Strategy (D-07 discretion)

Use a CSS container query or Tailwind responsive classes on the outer wrapper:

```
mobile  (<640px): max-width: 260px, centered
tablet  (640px+): max-width: 300px, centered
desktop (1024px+): max-width: 320px — matches current ScreenshotBlock max-width
```

Apply via `style="--frame-width: 260px"` CSS variable on the wrapper, overridden at breakpoints, so all inner dimension calculations scale proportionally via `calc()` if needed in future.

### Hero Frame Enhancement (D-05 discretion)

The hero DeviceFrame gets one enhancement beyond the standard frame:
- Subtle perspective tilt: `transform: perspective(1200px) rotateY(-4deg) rotateX(2deg)` — slight off-axis angle suggesting depth without distorting the screenshot content. Applied only to the hero wrapper div, not the component itself.
- Enhanced shadow for hero context: `0 40px 100px rgba(28,25,21,0.28), 0 8px 32px rgba(28,25,21,0.14)` — larger spread to match the larger hero viewport area.
- No additional CSS animations in Phase 6 — hero entrance choreography is Phase 7 scope.

### Image Optimization Contract (D-08, D-09)

| Page | loading | fetchpriority | widths prop |
|------|---------|---------------|-------------|
| Hero (dashboard.png) | eager | high | [320, 640, 960] |
| Feature pages | lazy (default) | auto (default) | [320, 640] |

The `widths` prop on Astro `<Image>` instructs the component to emit `srcset` with WebP and AVIF variants. Browser selects based on DPR and viewport width.

### Placeholder Styling (D-04)

Pages without screenshots (coaching, cold-start) render a styled placeholder inside the DeviceFrame's screen area:

- Background: linear gradient `from #E4E0DB to #CFCBC5` (surface-el to divider) — warm, neutral, on-brand
- Text: "Coming soon" in Label size (13px), weight 400, color `--color-text-3`, uppercase, letter-spacing 0.06em
- Positioned: `display: flex; align-items: center; justify-content: center`
- The placeholder occupies the full screen area (9/19.5 aspect ratio) inside the frame
