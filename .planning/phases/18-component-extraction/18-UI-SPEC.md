## Component Inventory

### New: LanguageSwitcher (inline in Header.astro + MobileMenu.astro)

No separate component file. Implemented inline following the Features dropdown pattern.

**Desktop trigger anatomy:**
```
[globe SVG 16×16] [locale code 13px/600] [chevron SVG 12×12]
```
- Globe: inline SVG, `aria-hidden="true"`, stroke `var(--color-text-2)` at rest
- Locale code: uppercase 2–3 character string (EN, 中文 abbreviated to ZH, FR) — rendered via translation utility
- Chevron: same 12×12 inline SVG already used in Features dropdown; rotates 180° when open

**Dropdown panel:**
- Width: 160px (fixed, not 300px like Features)
- Positioned: `top: calc(100% + 12px)`, `left: 50%`, `transform: translateX(-50%)`
- Uses existing `.nav-dropdown` CSS class — no new CSS needed for positioning/animation
- 3 locale items, each a plain `<a>` link (no icon, no description line)
- Active locale: `color: var(--color-accent)` + optional checkmark icon (16×16 inline SVG, right-aligned)

**Dropdown item anatomy:**
```
[locale name in native script]  [check SVG if active, else invisible]
```
- Padding: `var(--space-sm) var(--space-md)` — matches `.nav-dropdown-item`
- Uses `.nav-dropdown-item` hover background: `var(--color-surface-el)`

**Mobile trigger anatomy (between logo and hamburger):**
```
[globe SVG 16×16] [locale code] [chevron SVG 12×12]
```
- Same dropdown behavior as desktop (`.nav-dropdown` class)
- Touch target: min 44px height, achieved via flexbox centering
- Positioned in the header flex row: `order` between logo and hamburger via flex layout

**Interaction behavior (mirrors Features dropdown exactly):**
- Click trigger: toggles `.is-open` class on dropdown panel
- Hover (desktop): opens on `mouseenter`, closes after 200ms `mouseleave` delay
- Outside click: `document.addEventListener('click', ...)` closes dropdown
- Escape key: closes dropdown, returns focus to trigger button
- `aria-expanded` attribute toggled on trigger
- `aria-haspopup="listbox"` on trigger

**Animation:** `.nav-dropdown` CSS already handles opacity/transform/visibility transitions — no new keyframes needed:
```css
/* Already in global.css */
opacity: 0 → 1; transform: translateY(-8px) scale(0.96) → translateY(0) scale(1); 200ms ease
```

**Source:** 18-CONTEXT.md D-01, D-02, D-03; Header.astro existing Features dropdown

## Interaction States

| Element | State | Visual Treatment |
|---------|-------|-----------------|
| Switcher trigger | Rest | Globe(text-2) + locale code(text-1) + chevron(text-1) |
| Switcher trigger | Hover | `color: var(--color-accent)` on text (matches `.nav-link:hover`) |

## Globe Icon Specification

Inline SVG, 16×16px viewBox, no external sprite or icon package.

**Recommended path:** Simple circle with latitude/longitude lines (standard "globe" metaphor):
```svg
<svg width="16" height="16" viewBox="0 0 16 16" fill="none"
     stroke="currentColor" stroke-width="1.5" stroke-linecap="round"
     aria-hidden="true">
  <circle cx="8" cy="8" r="6.5"/>
  <path d="M8 1.5C8 1.5 5.5 4 5.5 8s2.5 6.5 2.5 6.5"/>
  <path d="M8 1.5C8 1.5 10.5 4 10.5 8s-2.5 6.5-2.5 6.5"/>
  <path d="M1.5 8h13"/>
</svg>
```

Color: `currentColor` (inherits from parent — rest: `var(--color-text-2)`, hover/open: `var(--color-accent)` via CSS cascade).

**Source:** 18-CONTEXT.md specifics; CONTEXT.md D-01 (SVG over emoji)

## Page Routing Contract

Language switching preserves current page path. Implementation: each locale item in the dropdown is an `<a>` tag with `href` computed as `getRelativeLocaleUrl(targetLocale, currentPath)`.

`currentPath` is derived from `Astro.url.pathname` stripped of its locale prefix, passed down to the switcher as a prop.

Pages without locale variants (e.g., `/methodology` until Phase 19): switching to `/zh/methodology` will 404 until that page exists. This is accepted per D-06 — correct routing from day one.

**Source:** 18-CONTEXT.md D-03, D-06
