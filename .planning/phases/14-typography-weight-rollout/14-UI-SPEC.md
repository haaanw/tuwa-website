## Implementation Notes for Executor

### Replacement Pattern

Replace all hardcoded `font-weight: {value}` with `font-weight: var(--weight-{tier})`:

**In inline `style=""` attributes:**
```html
<!-- Before -->
style="font-weight: 600;"

<!-- After -->
style="font-weight: var(--weight-heading);"
```

**In `<style>` blocks and global.css:**
```css
/* Before */
font-weight: 600;

/* After */
font-weight: var(--weight-heading);
```

### Zero Hardcoded Values Rule

After migration, running the following grep must return zero results (excluding global.css lines that define the tokens themselves):

```bash
grep -rn "font-weight: [0-9]" src/ --include="*.astro" --include="*.css"
```

The `@font-face` range declaration (`font-weight: 200 700`) is exempt ‚Äî that is not a design value.

### Visual Verification Targets

The executor must visually confirm these specific contrasts before marking the phase complete:

1. Hero headline (`Hero.astro`) reads visibly lighter than the subtitle paragraph beneath it ‚Äî 200 vs 500 contrast
2. StatsCounter animated numbers appear in ultralight editorial style ‚Äî unmistakably thinner than any surrounding text
3. Section headings on all 5 feature pages are visibly lighter than their body paragraphs
4. FAQ questions (600) read distinctly bolder than FAQ answers (500) ‚Äî subtle but observable
5. Footer section headers ("Features", "Resources", "Legal") read at body weight ‚Äî not as bold as buttons
