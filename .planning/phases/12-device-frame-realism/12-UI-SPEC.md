### Shadow System (DFRM-03)

4-layer graduated shadow stack — device sits on a desk under soft diffuse light. Values are Claude's discretion per CONTEXT.md.

```css
box-shadow:
  /* Layer 1 — tight contact shadow */
  0 2px 4px rgba(0,0,0,0.10),
  /* Layer 2 — close ambient */
  0 8px 16px rgba(0,0,0,0.12),
  /* Layer 3 — mid-distance falloff */
  0 20px 40px rgba(0,0,0,0.14),
  /* Layer 4 — wide soft bloom */
  0 40px 80px rgba(0,0,0,0.10),
  /* Inset edge highlights (preserve D-03) */
  inset 0 1px 0 rgba(255,255,255,0.10),
  inset 0 -1px 0 rgba(0,0,0,0.30);
```

Note: The current 3-layer shadow (2/4px, 8/24px, 24/48px) upgrades to 4 layers with a wider bloom at 40/80px. Opacity values (0.10–0.14) keep it premium, not dramatic. Tune visually against `#F4F1ED` travertine background.

### Screen Inset Shadow (DFRM-04)

Inset shadow on the screen area div to make display appear recessed into bezel:

```css
box-shadow: inset 0 2px 6px rgba(0,0,0,0.15), inset 0 0 1px rgba(0,0,0,0.08);
```

Keep subtle enough that it does not noticeably darken screenshot edges — verification criterion: screenshot colors read correctly in the top 10% of the frame area.

## Component Inventory

### DeviceFrame.astro

Existing component to be enhanced in-place. No new components introduced.

**Structural changes:**
- Migrate bezel `style=""` inline styles to `.device-frame` CSS class in `global.css` (D-12 cleanup, CONTEXT.md discretion)
- Migrate Dynamic Island inline styles to `.device-island` CSS class
- Migrate screen area inline styles to `.device-screen` CSS class
- Keep component props API identical — callers (Hero.astro, FeaturePageLayout.astro, CoachingPageLayout.astro) must not need changes

**Side button changes (.device-frame::before — left side):**

Current: `::before` = volume up only + `box-shadow: 0 36px 0` for volume down (2 buttons on left)

Target: 3 buttons on left (action button + volume up + volume down) using `box-shadow` stacking:
```css
/* ::before renders action button + volume up + volume down via box-shadow offsets */
.device-frame::before {
  content: '';
  position: absolute;
  left: -3px;
  top: 80px;           /* action button position */
  width: 3px;
  height: 24px;        /* action button height */
  background: #2A2A2A;
  border-radius: 2px 0 0 2px;
  box-shadow:
    0 30px 0 #2A2A2A,  /* volume up: offset ~30px below action button, height 32px implied by next offset */
    0 66px 0 #2A2A2A;  /* volume down: offset ~66px below action button */
}
```

Note: `box-shadow` with `0 Npx 0` creates same-size copies at vertical offset. Use `height` on the pseudo-element to control each button's height. Volume up and down share the same height as the `::before` element — adjust with separate pseudo-element strategy if distinct heights are required.

**Power button (right side, ::after):** keep existing proportions.

### Screenshot Fit Fix (DFRM-05)

Root cause: border-radius mismatch between screen area container (40px) and the `<Image>` component inside it. The image may not fill the full border-radius clip correctly.

Fix approach:
1. Ensure `.device-screen` has `overflow: hidden` and `border-radius: 40px` (existing — verify)
2. Image must be `width: 100%; height: 100%; object-fit: cover; display: block` (existing — verify)
3. Remove any `padding` or `border` on the screen area that would create a gap
4. Confirm exported screenshots match 393:852 ratio exactly — if they don't, the cover fill will show letterboxing

Verification: At 260px frame width, no gap visible between image and frame inner bezel edge.

### Responsive Scaling (DFRM-01, DFRM-02, DFRM-06)

Frame breakpoints are fixed by existing Tailwind utility classes — do not change:
- 260px: mobile (`max-w-[260px]`)
- 300px: sm (`sm:max-w-[300px]`)
- 320px: lg (`lg:max-w-[320px]`)

Dynamic Island must scale with frame:

```css
.device-island {
  width: 32%;           /* ~32% of screen width — matches real iPhone 15 Pro 126/393 */
  aspect-ratio: 2.7;   /* ~2.7:1 — matches 126:37.33 hardware ratio */
}
```

At 260px frame (approx 236px screen after padding): island = ~75px wide × ~28px tall
At 320px frame (approx 296px screen after padding): island = ~95px wide × ~35px tall
