# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

### Font Loading Contract

- Files: `AlpinoVariable.woff2` (variable axis preferred) or weight-specific woff2 files for Regular (400) and Semibold (600)
- Declare `@font-face` blocks in `src/styles/global.css` with `font-display: swap`
- Add `<link rel="preload" as="font" type="font/woff2" crossorigin>` in `<head>` for the Regular weight file
- No FOUT, no CLS regression (FOUND-02 success criterion)

|-------|-------|-------------|-------|
| xs | 4px | `--space-xs` | Icon gaps, hairline margins, inline label-to-icon |
| sm | 8px | `--space-sm` | Compact element spacing, nav link padding |

| lg | 24px | `--space-lg` | Card internal vertical padding, standard section gaps |
| xl | 32px | `--space-xl` | Layout column gaps |
| 2xl | 48px | `--space-2xl` | Major section breaks |
| 3xl | 64px | `--space-3xl` | Page-level top/bottom breathing room |

Exceptions:
- Navigation height: 64px (aligns to 3xl; provides a substantial header with breathing room)

- Footer internal padding: 48px top, 32px bottom

All other values must be from the token scale. No magic numbers.

## Typography

Source: App DESIGN.md type scale, adapted for web with Alpino replacing DM Sans. Web context adds a Display size for hero use in Phase 2; declared here so the token exists in global.css from day one.

| Role | Size | Weight | Line Height | Letter Spacing | CSS Class / Notes |
|------|------|--------|-------------|----------------|-------------------|
| Display | 48px | 600 (Semibold) | 1.1 | -0.02em | Reserved for Phase 2 hero headline — token declared now |
| Heading | 28px | 600 (Semibold) | 1.2 | -0.01em | Page titles, section headers |
| Body | 16px | 400 (Regular) | 1.6 | 0 | Navigation links, footer text, paragraph copy |
| Label | 13px | 400 (Regular) | 1.4 | +0.06em (all-caps contexts) | Micro labels, nav supplemental, legal fine print |

Weights used: Regular (400) and Semibold (600) only. No other weights. This preserves the app's "hierarchy through size, not weight variation" principle while giving the web one semibold tier for headings and CTAs.

Numeric displays: apply `font-variant-numeric: tabular-nums` to any metric or number display (consistent with app's `.monospacedDigit()` rule).

### Accent and Interactive

Decision D-02: Introduce a distinct contrasting accent for web CTAs. The app's travertine accent (`#7A6E5C`) blends into the warm surface — it cannot function as an action color on the web. This spec selects **deep forest green** as the web CTA accent. Rationale: sits on the cool axis opposite the warm travertine (thermal contrast, not just value contrast); connects to the app's zone-optimal color family (`#3E5C49`); conveys credibility, performance, and precision without aggression. Derived by deepening zone-optimal for higher contrast on light backgrounds.

| Role | Hex | CSS Variable | Usage |
|------|-----|-------------|-------|
| Accent (10%) | `#2B5240` | `--color-accent` | CTA buttons only (see reserved list below) |
| Accent hover | `#1E3D2F` | `--color-accent-hover` | CTA button hover/active state |
| Accent text on accent | `#F4F1ED` | `--color-accent-fg` | Text on accent-colored buttons |
| Travertine accent | `#7A6E5C` | `--color-brand-accent` | Brand mark, decorative use only — never on interactive elements |
| Destructive | `#6E3A3A` | `--color-destructive` | Destructive actions only — none in Phase 1 |

Accent (`--color-accent`) is reserved for exactly these elements:

1. "Get the App" CTA button in the navigation header
2. App Store badge call-to-action in the footer
3. No other elements in Phase 1

The travertine mid-tone (`--color-brand-accent`) may appear on the wordmark or brand decoration but never on any interactive element.

### 60/30/10 Confirmation

- 60%: `#F4F1ED` — page background (the dominant material, like travertine stone)
- 30%: `#EDEAE6` — nav bar, footer, cards (lifted surface)
- 10%: `#2B5240` — the two CTA elements above; accent is precise and withheld

## Layout Contract

### Header

- Height: 64px
- Position: D-08 is Claude's Discretion. Recommendation: **sticky** (position: sticky, top: 0, z-index above content). Rationale: The "Get the App" CTA must remain visible on marketing pages where the visitor's intent to download may be triggered at any scroll depth. Standard marketing site conversion best practice. Background: `--color-surface` at 95% opacity with `backdrop-filter: blur(8px)` for a subtle depth cue without the strict app "no shadows" rule (D-03 permits web flourishes).
- Desktop: logo left, nav links center-right, "Get the App" button rightmost
- Mobile: logo left, hamburger right — nav links hidden until menu opens
- Mobile breakpoint: < 768px

### Footer

- Multi-column grid (D-05): 3 columns on desktop (Product, Resources, Legal), stacked single-column on mobile
- App Store badge: prominent, above the column grid or spanning full width on mobile
- Copyright: below column grid, left-aligned on desktop

### URL Structure

D-07 is Claude's Discretion. Recommendation: **`/features/[slug]`** over flat `/<slug>`. Rationale: Grouped namespace signals content hierarchy to Google; protects the flat namespace for top-level brand pages (Privacy, Terms, Support, Blog); prevents accidental slug collisions; aligns with how `@astrojs/sitemap` generates crawlable paths. Established in Phase 1 as a routing convention even though content pages are Phase 3.

### Responsive Breakpoints

| Name | Width | Notes |
|------|-------|-------|
| Mobile | < 768px | Single-column, hamburger menu |
| Tablet | 768px – 1023px | Two-column grid available, full nav |
| Desktop | >= 1024px | Full layout, multi-column footer |

### Border Radius

Per D-03: 4–8px. Specific values:
- CTA button: 6px
- Mobile menu drawer: 0 (full-height overlay)
- Card containers (footer panels): 6px
- No other rounded elements in Phase 1

### Shadows

Per D-03: subtle is permitted. Specific values:
- Header on scroll: `box-shadow: 0 1px 0 var(--color-divider)` (hairline bottom border via shadow — consistent with app divider approach)
- CTA button hover: `box-shadow: 0 2px 8px rgba(43, 82, 64, 0.20)` (accent-tinted, subtle)
- No other shadows in Phase 1

## Motion Contract

Phase 1 motion scope: navigation interactions only. No scroll animations yet (Phase 2).

| Interaction | Duration | Easing | Notes |
|------------|---------|--------|-------|
| Mobile menu open | 200ms | ease-out | Drawer slides down from top or fades in |
| Mobile menu close | 150ms | ease-in | Faster on exit — standard UX convention |
| Nav link hover | 100ms | linear | Color transition on text: text-1 → accent |
| CTA button hover | 100ms | linear | Background: accent → accent-hover |
| CTA button active/press | 80ms | linear | Scale: 1 → 0.97 (subtle press feedback) |

No spring physics. No bounce. Consistent with app motion principles.

## Accessibility Contract

- All nav links and CTA button: visible focus ring using `outline: 2px solid var(--color-accent); outline-offset: 2px`
- Mobile menu: focus trap when open; Escape key closes; aria-expanded on hamburger button
- Header nav: `<nav>` with `aria-label="Main navigation"`
- Footer nav: `<nav>` with `aria-label="Footer navigation"`
- App Store badge: meaningful alt text (see Copywriting Contract above)
- Heading hierarchy: `<h1>` appears once per page in the main content slot (not in header/footer)
- Color contrast: all text tokens verified against background tokens
  - `#1C1915` on `#F4F1ED`: ~14:1 (exceeds WCAG AAA)
  - `#696560` on `#F4F1ED`: ~5.4:1 (WCAG AA pass)
  - `#F4F1ED` on `#2B5240` (CTA button): ~8.2:1 (WCAG AAA)
