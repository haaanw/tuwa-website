
2. Rewrite `astro.config.mjs` to the following exact content:
```javascript
// @ts-check
import { defineConfig, fontProviders } from "astro/config";
import tailwindcss from "@tailwindcss/vite";
import mdx from "@astrojs/mdx";
import sitemap from "@astrojs/sitemap";

export default defineConfig({
  site: "https://tuwa.app",
  fonts: [
    {
      provider: fontProviders.fontshare(),
      name: "Alpino",
      cssVariable: "--font-alpino",
      weights: ["400", "600"],
      styles: ["normal"],
      display: "swap",
      fallbacks: ["system-ui", "sans-serif"],
      optimizedFallbacks: true,
    }
  ],
  integrations: [mdx(), sitemap()],
  vite: {
    plugins: [tailwindcss()],
  },
});
```

CRITICAL per CLAUDE.md and RESEARCH.md:
- `tailwindcss()` goes in `vite.plugins`, NOT in `integrations` — putting it in integrations causes silent style failures
- `site: "https://tuwa.app"` is required for @astrojs/sitemap to work
- Do NOT add `@astrojs/cloudflare` adapter — it conflicts with static output
- Fonts API with `fontProviders.fontshare()` handles Alpino — if the build fails to resolve the font, the fallback is documented in Plan 01 Task 2 (manual @font-face in global.css)

3. After install, verify the build succeeds:
```bash
npx astro build
```
If the Fonts API fails to resolve "Alpino" from Fontshare, remove the `fonts` array from astro.config.mjs and proceed — the font fallback will be handled in Task 2's global.css with manual @font-face.
