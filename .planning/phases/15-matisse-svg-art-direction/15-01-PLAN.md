**Step 1: Create `scripts/generate-blobs.mjs`**

Create a Node ESM script that generates 7 biomorphic blob SVG path strings using the Catmull-Rom-to-cubic-Bezier algorithm from RESEARCH.md Pattern 1. Per D-01, D-02, D-03:

- Generate 7 distinct shapes (abstract blobs, no figurative forms)
- Use `numPoints = 8` and `variance` between 0.30–0.45 per shape
- Use a fixed seed per shape for reproducibility (e.g., seeds: 0.1, 0.25, 0.4, 0.55, 0.7, 0.85, 0.95)
- ViewBox: 200x200 (centerX=100, centerY=100, baseRadius=70)
- Output: Write a temporary SVG file `scripts/blobs-raw.svg` containing all 7 paths inside a single `<svg>` with flat structure (no nested `<g>`)
- Also print each path's `d` attribute to stdout for reference

The `generateBlob` function:
```javascript
function generateBlob({ numPoints = 8, centerX = 100, centerY = 100, baseRadius = 70, variance = 0.35, seed = 0.5 }) {
  const points = [];
  const angleStep = (Math.PI * 2) / numPoints;
  for (let i = 0; i < numPoints; i++) {
    const theta = i * angleStep;
    const r = baseRadius * (1 - variance + variance * pseudoRandom(seed + i));
    points.push({ x: centerX + Math.cos(theta) * r, y: centerY + Math.sin(theta) * r });
  }
  return pointsToCubicBezierPath(points, true);
}
```

Use a simple seeded pseudo-random: `function pseudoRandom(s) { const x = Math.sin(s * 9999.9) * 9999.9; return x - Math.floor(x); }`

The `pointsToCubicBezierPath` function converts points to a closed cubic Bezier SVG path string using the Catmull-Rom formula (see RESEARCH.md Pattern 1 for exact implementation).

**Step 3: Create `src/components/MatisseShape.astro`**

A props-driven component that renders a single SVG `<path>` element. Props interface:

```typescript
interface Props {
  shapeId: 'blob-1' | 'blob-2' | 'blob-3' | 'blob-4' | 'blob-5' | 'blob-6' | 'blob-7';
  fill?: string;       // hex color, default '#2B5240'
  opacity?: number;    // 0-1, default 1
  transform?: string;  // SVG transform attribute, e.g. 'translate(80, 40) rotate(-12) scale(1.5)'
  parallaxSpeed?: 'slow' | 'fast' | 'none';  // default 'none'
  animateDelay?: string; // e.g. '80ms', default '0ms'
  class?: string;      // additional CSS classes
}
```

Hardcode the 7 SVGO-optimized path `d` strings in a `const SHAPES: Record<string, string>` map inside the frontmatter. The component renders:

```html
<path
  class:list={["matisse-shape", Astro.props.class]}
  fill={fill}
  opacity={opacity}
  d={SHAPES[shapeId]}
  transform={transform}
  data-parallax-slow={parallaxSpeed === 'slow' ? '' : undefined}
  data-parallax-fast={parallaxSpeed === 'fast' ? '' : undefined}
  style={`animation-delay: ${animateDelay}`}
/>
```

No wrapping `<svg>` — this component is meant to be placed inside an existing `<svg>` element. The parent `<svg>` provides the viewBox.

/* Parallax — progressive enhancement only (ART-05) */
/* Chrome 115+, Safari 26+ only; Firefox behind flag; degrades to static shapes */
@supports (animation-timeline: scroll()) {
  @media (prefers-reduced-motion: no-preference) {
    /* Hero frieze shapes: dual animation (entrance + parallax) */

      animation-fill-mode: both, both;
      animation-timeline: auto, scroll(root block);
      animation-range: normal, entry 0% exit 100%;
    }
    .matisse-frieze .matisse-shape[data-parallax-fast] {
      animation-name: matisse-shape-enter, matisse-parallax-fast;
      animation-duration: 600ms, auto;
      animation-timing-function: cubic-bezier(0.16, 1, 0.3, 1), linear;
      animation-fill-mode: both, both;
      animation-timeline: auto, scroll(root block);
      animation-range: normal, entry 0% exit 100%;

    @keyframes matisse-parallax-slow {
      from { transform: translateY(0) translateZ(0); }
      to   { transform: translateY(-12px) translateZ(0); }

}
