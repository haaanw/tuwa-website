## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4F1ED` (`--color-bg`) | Page background — travertine off-white |
| Secondary (30%) | `#EDEAE6` (`--color-surface`) | Nav bar, footer, cards |
| Accent (10%) | `#2B5240` (`--color-accent`) | CTA buttons AND Matisse shape fills (hero frieze primary color) |
| Matisse complement A | `#8B5E3C` (terracotta warm) | 1–2 hero frieze shapes only — never on interactive elements |
| Matisse complement B | `#A89060` (soft gold/sand) | 1–2 hero frieze shapes only — never on interactive elements |
| Feature page shapes | `#2B5240` at 15–25% opacity | Decorative shapes on feature deep-dive pages (lighter treatment) |
| Destructive | `#6E3A3A` (`--color-destructive`) | Destructive actions only (none in this phase) |

**Accent reserved for:**
- CTA buttons (`Download on the App Store`, hero App Store badge link)
- Hero frieze Matisse shapes (primary fill — solid/opaque)
- Feature page Matisse shapes (fill at 15–25% opacity)

**Matisse color contract (from CONTEXT.md D-11 and D-12/D-13):**
- Hero shapes: solid fills — one shape per complementary color (2–3 shapes total in terracotta/gold), remaining shapes use accent green `#2B5240`
- Feature page shapes: `#2B5240` at 15–25% opacity only (no complementary colors on subpages — maintains visual hierarchy)
- If terracotta `#8B5E3C` or sand `#A89060` clash on travertine at runtime, fall back to charcoal `#1C1915` at 8% opacity per D-14

**Reasoning for hue selection:** Travertine `#F4F1ED` is a warm, slightly yellow-beige. Terracotta `#8B5E3C` harmonizes on the warm axis. Sand/gold `#A89060` provides middle-ground warmth. Both read as paper-cut-out pigments consistent with Matisse's gouache palette. Cool blue was rejected — it would fight the warm background.

Source: `src/styles/global.css` `:root` color tokens + CONTEXT.md D-11, D-12, D-13, D-14

## Matisse Shape Visual Contract

This section is unique to Phase 15 and required by ART-01 through ART-06.

### Shape Vocabulary (ART-01, D-02, D-03)

- **Count:** 5–7 distinct biomorphic blob shapes
- **Style:** Abstract only — no figurative forms (no leaves, bodies, teardrops, or recognizable objects)
- **Authoring:** Programmatic Bezier curves in code — no Figma/Inkscape source required (D-01)
- **Format:** Inline SVG inside Astro components (consistent with established codebase pattern)
- **Optimization:** SVGO-optimized path data to keep DOM node count within Lighthouse budget (ART-03)
- **ViewBox:** Each shape authored in a normalized 100×100 or 200×200 viewBox; scaled via CSS `width`/`height`

### Hero Frieze (ART-02, D-05 through D-08)

- **Container:** `.matisse-frieze` (existing stub — `position: absolute; inset: 0; overflow: hidden; pointer-events: none`)
- **Placement:** Background layer behind hero content (headline, device frame, badge render on top via `z-index`)
- **Band:** Horizontal strip clustered around the device frame vertical position
  - Height: 240px (centered vertically within the hero section)
  - Width: 100vw (edge-to-edge, clipped by `overflow: hidden`)
- **Shape arrangement:** 5–7 shapes placed in a continuous left-to-right strip with deliberate spacing. Shapes may overlap by 10–20% for a "placed" composition feel. No shape should appear isolated from the others.
- **Scale variation:** Shapes range from 80px to 180px wide — never uniform. At least 2 shapes must be significantly larger than the others.
- **Rotation:** Each shape rotated between -15deg and +20deg for organic variety. No two adjacent shapes share the same rotation.

### Feature Page Decoration (ART-04, D-09, D-10)

Two approaches to implement and compare before committing:

**Approach A — Small cluster:**
- 2–3 shapes from the vocabulary, grouped at top-right or bottom-left of the feature page header
- Size: 60–120px each
- Opacity: `0.18` (18% — lighter than hero)
- CSS class: `.matisse-decoration .matisse-cluster`

**Approach B — Section divider:**
- 3–4 shapes arranged as a narrow horizontal band between two content sections
- Height: 80px band
- Opacity: `0.15`
- CSS class: `.matisse-decoration .matisse-divider`

Each feature page uses exactly ONE shape from the vocabulary (not repeated across pages per D-10). Assign shapes by page to ensure each deep-dive page has a unique identifier shape.

### Parallax Animation (ART-05)

- **Mechanism:** CSS `animation-timeline: scroll()` (established in Phase 11 `.matisse-shape` stub — `will-change: transform` already set)
- **Direction:** Vertical — shapes drift upward as user scrolls down (negative Y translate)
- **Intensity:** 20px total range (translateY from 0 to -20px over the full scroll range)
- **Rate differential:** Different shapes move at 0.6× and 0.8× of the page scroll rate — never 1:1 (creates depth)
- **Reduced motion:** All parallax disabled under `prefers-reduced-motion: reduce` (existing guard at `global.css` lines 731–738)

Implementation note: Use `animation-timeline: scroll(root block)` with `animation-range: entry 0% exit 100%` per CSS Scroll-Driven Animations spec.

### Entrance Animation (ART-06)

- **Trigger:** IntersectionObserver — same `data-animate` pattern as `src/layouts/BaseLayout.astro` line 39
- **Style:** Fade-in + translate-up — consistent with existing scroll-reveal pattern
  - Initial state: `opacity: 0; transform: translateY(24px)`
  - Animated state: `opacity: 1; transform: translateY(0)`
- **Duration:** 600ms
- **Easing:** `cubic-bezier(0.16, 1, 0.3, 1)` (spring-like ease-out — matches existing scroll-reveal feel)
- **Stagger:** 80ms per shape (first shape fires at 0ms, second at 80ms, etc.)
- **Hero frieze:** Entrance fires on page load (not scroll — frieze is above the fold)
- **Feature page shapes:** Entrance fires when shape enters viewport (standard IO pattern)
- **Reduced motion:** `animation: none !important` (existing guard covers this)

## DOM Budget Contract (ART-03)

Lighthouse flags "Avoid an excessive DOM size" at 1,500 nodes. Phase 15 must not trigger this warning.

- Hero frieze: maximum 7 SVG elements × ~3 nodes each = 21 nodes added
- Feature page decorations: maximum 3 SVG elements × ~3 nodes each = 9 nodes added
- Total Phase 15 DOM addition: ≤ 30 nodes
- Verification: Run Lighthouse DOM Size audit before and after; confirm no warning appears

SVGO optimization requirements:
- Strip all metadata, comments, editor data
- Merge path segments where possible
- Each shape path: ≤ 8 Bezier control points (≤ 1 SVG `<path d="...">` element per shape)

## Animation Performance Contract

All animations must pass:
- No layout-triggering properties (no `width`, `height`, `margin`, `padding` in animations)
- Only `transform` and `opacity` animated (compositor-only — 60fps on mid-range hardware)
- `will-change: transform, opacity` pre-declared (done in Phase 11 stub)
- `translateZ(0)` pre-declared (done in Phase 11 stub — GPU layer promotion)
