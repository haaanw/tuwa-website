**Step 1 — Download the variable WOFF2 from Fontshare CDN:**

First, create the `public/fonts/` directory if it does not exist. Then download the variable WOFF2 file.

The variable font URL must be extracted from Fontshare's CSS API endpoint. Run:

```bash
mkdir -p public/fonts
# Extract the WOFF2 URL from the Fontshare CSS endpoint for the variable font (style number 1)
WOFF2_URL=$(curl -s "https://api.fontshare.com/v2/css?f[]=general-sans@1" | grep -o '//cdn.fontshare.com[^)]*\.woff2')
curl -sL "https:${WOFF2_URL}" -o public/fonts/GeneralSans-Variable.woff2
# Verify file is non-empty and is a valid WOFF2 (starts with wOF2 magic bytes)
file public/fonts/GeneralSans-Variable.woff2
```

The file should be ~80-120KB (a single variable WOFF2 covering weight axis 200-700).

**Step 2 — Add @font-face declaration to global.css:**

Insert the following block at the VERY TOP of `src/styles/global.css`, BEFORE the existing `@import "tailwindcss"` line (or before the first `:root` block if no import exists). This replaces the Astro Font API's auto-generated `@font-face`:

/* ===== Phase 11: Self-Hosted Variable Font (TYPO-01) ===== */

**Edit 1 — View transition block in global.css:**

Insert the following block AFTER the `.section-spaced` responsive media query closing brace and BEFORE `.nav-logo`. Add a blank line before and after:

```css
/* ===== Phase 11: View Transitions (IXPN-01) ===== */
/* MPA browser-native crossfade. Both outgoing and incoming documents need this rule.  */
/* global.css loads on every page via BaseLayout.astro — single rule covers all pages. */
/* No @media wrapper — browser suppresses transitions automatically for prefers-reduced-motion. */
@view-transition {
  navigation: auto;
}

/* Override browser default 250ms crossfade -> 200ms (D-07). Easing: ease-out. */
::view-transition-old(root) {
  animation: 200ms ease-out both crossfade-out;
}
::view-transition-new(root) {
  animation: 200ms ease-out both crossfade-in;
}

@keyframes crossfade-out {
  from { opacity: 1; }
  to   { opacity: 0; }
}
@keyframes crossfade-in {
  from { opacity: 0; }
  to   { opacity: 1; }
}
```

Per D-06: native CSS `@view-transition { navigation: auto }` — NOT `<ClientRouter>` or `<ViewTransitions />`. Per D-07: 200ms duration. Per D-08: global crossfade only, no per-element named transitions. Easing: ease-out (Claude's discretion — front-loaded opacity decay feels snappy at 200ms).

CRITICAL: `@view-transition` must be at the top level of the stylesheet — NOT nested inside any `@media` query. The browser handles `prefers-reduced-motion` suppression internally for view transitions.

CRITICAL: Do NOT import or add `<ClientRouter>` or `<ViewTransitions />` from `astro:transitions` anywhere. The native CSS approach preserves IntersectionObserver compatibility (IXPN-02).
