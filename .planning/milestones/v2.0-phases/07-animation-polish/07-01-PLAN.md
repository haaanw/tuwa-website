```javascript
document.querySelectorAll('[data-animate]').forEach(function (el) {
  // ANIM-03 stagger: read data-animate-delay, set as inline animationDelay
  // Required: CSS attr() with type hints not supported in Firefox/Safari as of May 2026
  var delay = el.getAttribute('data-animate-delay');
  if (delay) {
    el.style.animationDelay = delay;
  }
  observer.observe(el);
});
```

**global.css** — Add the following block AFTER the existing `@media (prefers-reduced-motion: reduce)` block (after line 103). Do NOT modify anything above line 103.

```css
/* Hero entrance animations — ANIM-04 */
/* Fires immediately on page load. No IntersectionObserver. No .js-enabled gate. */
/* fill-mode: both applies 'from' state before first paint — prevents FOUC */
@media (prefers-reduced-motion: no-preference) {
  .hero-headline {
    animation: fade-up 400ms ease-out 0ms both;
  }
  .hero-subtitle {
    animation: fade-up 400ms ease-out 150ms both;
  }
  .hero-device {
    animation: hero-device-enter 500ms ease-out 350ms both;
  }

  /* hero-device-enter: compound transform preserves perspective tilt + scale payoff */
  @keyframes hero-device-enter {
    from {
      opacity: 0;
      transform: perspective(1600px) rotateY(-2deg) rotateX(1deg) translateY(20px) scale(0.96);
    }
    to {
      opacity: 1;
      transform: perspective(1600px) rotateY(-2deg) rotateX(1deg) translateY(0) scale(1);
    }
  }
}
```

Key details:
- Uses the existing `fade-up` keyframe for headline and subtitle (already defined above at line 94).
- New `hero-device-enter` keyframe for device mockup with compound transform preserving the perspective(1600px) rotateY(-2deg) rotateX(1deg) tilt.
- `both` fill-mode (not `forwards`) to prevent FOUC — elements start at `from` state before animation begins.
- `@keyframes hero-device-enter` at top level of the `@media` block, not nested inside a selector.
- No `.js-enabled` gate — hero animates via CSS regardless of JS state. Under reduced-motion, classes are inert (no animation declaration exists).
