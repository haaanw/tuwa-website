## Interaction Contract

Source: CONTEXT.md decisions D-04 through D-10.

### Drag-to-Rotate

- **Pointer tracking:** `pointerdown` â†’ `pointermove` â†’ `pointerup` (works for mouse and touch via Pointer Events API)
- **Rotation calculation:** track cumulative delta angle from pointer start angle relative to wheel center
- **Snap behavior:** on `pointerup`, snap to nearest of 5 feature positions (72Â° apart). Snap uses CSS transition `transform 200ms ease-out` (Phase 7 motion direction: precise & swift, 150â€“200ms ease-out)
- **Momentum/inertia:** on release, calculate pointer velocity (px/ms over last 3 samples). Apply velocity-based overshoot then snap back. Overshoot clamp: max 30Â° beyond nearest segment. Inertia decay: 300ms ease-out. If `prefers-reduced-motion` is set â€” no momentum, immediate snap.
- **Coordinate math:** use `Math.atan2(dy, dx)` relative to wheel center for angle tracking. Convert drag delta to rotation delta in degrees.

### Click-to-Select

- Each segment is a `<button>` element (or `<a>` if linking directly â€” use `<button>` for segment selection, reserve `<a>` for center preview CTA only)
- Clicking a segment: rotate wheel to place that segment at the "top" (12 o'clock) position. Transition: `transform 250ms ease-out`
- If `prefers-reduced-motion`: no rotation animation, segment activates instantly (state updates immediately, no CSS transition fires)

### Center Preview

- Updates on each segment activation (drag snap or click)
- Content swap: crossfade via CSS opacity transition `150ms ease-out` (old content fades out over 75ms, new content fades in over 75ms). If `prefers-reduced-motion`: instant swap, no opacity transition.
- Center disc is always visible; "loading" state not needed (all 5 feature data sets are inline in the component)
- Center disc contains: icon (24px, `--color-accent`), feature title (weight 600), description (weight 400, max 2 lines via `overflow: hidden; display: -webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical`), "Explore â†’" link

### Keyboard Navigation

- `Tab` moves focus to the wheel widget (focus trap not needed â€” it's a navigation component, not a modal)
- `ArrowLeft` / `ArrowRight`: rotate to previous/next segment
- `Enter` / `Space`: navigate to the focused feature's deep-dive page (same as clicking the "Explore â†’" link)
- `aria-label` on the wheel container: `"Feature navigation wheel"`
- Each segment button: `aria-pressed="true/false"` for selected state
- Center preview region: `role="region"` with `aria-live="polite"` so screen readers announce feature changes on rotate

### Entrance Animation

Hooks into existing `AnimationController` IntersectionObserver via `data-animate` attribute on the wheel wrapper.

- Entrance: `fade-up` (same keyframe as all other `data-animate` elements â€” `opacity 0â†’1`, `translateY 16pxâ†’0`, `400ms ease-out`)
- Delay: `data-animate-delay="0ms"` â€” no stagger needed; the wheel enters as a single unit
- Wheel elements (rim segments) do NOT have individual `data-animate` â€” the whole component enters together
- If `prefers-reduced-motion`: entrance animation suppressed via existing `@media (prefers-reduced-motion: no-preference)` gate
