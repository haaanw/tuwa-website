# Phase 08.1: FeatureGrid Click Wheel - Context

**Gathered:** 2026-05-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the current bento grid FeatureGrid with an iPod-style click wheel interaction. The 5 feature cards are arranged around a circular wheel. Users rotate the wheel (drag or click) to browse features, with the selected feature previewed in the center. Does not add new features, pages, or change content — only transforms the FeatureGrid layout and interaction model.

</domain>

<decisions>
## Implementation Decisions

### Wheel Visual Design
- **D-01:** Literal circle wheel inspired by the iPod click wheel — a visible circular ring with feature names/icons placed around the circumference. Not a 3D carousel or pie menu.
- **D-02:** Each feature occupies a segment on the ring with its icon and name visible at rest.
- **D-03:** The currently selected/active feature is visually highlighted on the ring (accent color, scale, or glow — Claude's Discretion on exact treatment).

### Interaction Model
- **D-04:** Both drag-to-rotate AND click-to-select. Drag along the circle to rotate smoothly (pointer tracking with JS). Click any feature name on the rim to jump to it. Full iPod-like experience.
- **D-05:** Center preview on selection — selected feature shows icon + title + description in the center of the wheel. Center area acts as a link to the feature's deep-dive page.
- **D-06:** Rotation should feel smooth and physical — momentum/inertia on drag release. Duration and easing within the "precise & swift" motion direction from Phase 7.

### Mobile Behavior
- **D-07:** Same wheel on mobile, scaled down. Touch drag works naturally. No fallback to bento grid or list — the wheel is the experience at all breakpoints.
- **D-08:** Touch events mapped to the same rotation logic as mouse drag.

### Animation & Motion
- **D-09:** Rotation transitions respect `prefers-reduced-motion` — when reduced motion is enabled, clicking a segment jumps to the selection without rotation animation.
- **D-10:** Claude's Discretion — entrance animation for the wheel itself (fade-in, scale-up, or other). Should integrate with existing AnimationController IntersectionObserver pattern.

### Claude's Discretion
- Wheel diameter and sizing at each breakpoint
- Exact highlight treatment for active segment (color, scale, glow)
- Momentum/inertia physics parameters for drag release
- Center preview layout and typography
- SVG vs CSS approach for the wheel ring visual
- Entrance animation style

</decisions>

## Existing Code Insights

### Reusable Assets
- `AnimationController` in BaseLayout — IntersectionObserver with `data-animate` + `.is-visible`. Wheel entrance can use same trigger.
- Feature card data (icons, labels, titles, descriptions, URLs) already in FeatureGrid.astro — extract into data structure for wheel.

### Established Patterns
- `data-animate` attribute triggers observer → `.is-visible` class → CSS animation
- `@media (prefers-reduced-motion: no-preference)` gating on all animations
- `is:inline` script pattern for browser-side JS in Astro
- Inline `style` attributes with CSS custom properties

### Integration Points
- `FeatureGrid.astro` — complete rewrite from bento grid to wheel
- `global.css` — remove bento grid CSS, add wheel CSS
- `index.astro` — no change needed (FeatureGrid component reference stays the same)
- `BaseLayout.astro` — AnimationController stays; wheel may need its own `is:inline` script for drag/rotation logic

<specifics>
## Specific Ideas

- **iPod click wheel reference**: The original iPod had a flat circular touchpad with menu items around the edge and a center select button. The digital version should evoke that same feeling — rotate to browse, center to preview/select.
- **5 features around the circle**: Recovery Scoring, Workload Tracking, Smart Templates, Cold-Start Onboarding, Coach + Athlete — evenly spaced around the ring.
- **Center as CTA**: The center area shows the selected feature's preview and doubles as a navigation link to the deep-dive page.

</specifics>
