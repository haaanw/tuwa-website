## Animation Behavior Contract

This section is the primary deliverable of this phase. It defines the exact CSS and JS behavior that the executor must implement.

### JS-Gating Rule (CONTEXT.md D-02)

**Requirement:** Elements with `[data-animate]` must be fully visible when JavaScript is disabled.

**Implementation contract:**
- The AnimationController script adds `.js-enabled` to `<html>` as its first action, before querying any elements.
- All CSS rules that set `opacity: 0` or `transform: translateY(16px)` on `[data-animate]` elements MUST be scoped under `.js-enabled [data-animate]`.
- Without `.js-enabled` on `<html>`, no `[data-animate]` element is hidden. Content is visible by default.

**Before (wrong — hides content without JS):**
```css
@media (prefers-reduced-motion: no-preference) {
  [data-animate] {
    opacity: 0;
    transform: translateY(16px);
  }
}
```

**After (correct — JS-gated):**
```css
@media (prefers-reduced-motion: no-preference) {
  .js-enabled [data-animate] {
    opacity: 0;
    transform: translateY(16px);
  }
  .js-enabled [data-animate].is-visible {
    animation: fade-up 400ms ease-out forwards;
  }
}
```

### Reduced-Motion Rule (CONTEXT.md D-03)

**Requirement:** Users with `prefers-reduced-motion: reduce` must see zero motion. Elements render as if `[data-animate]` were not present.

**Current behavior (wrong):** The existing `@media (prefers-reduced-motion: reduce)` block applies `opacity: 0` and a transition — this is motion and it hides content for a reduced-motion user before JS runs.

**Correct behavior:**
- The `@media (prefers-reduced-motion: reduce)` block in `global.css` must contain NO rules that apply `opacity: 0`, `transform`, or `transition` to `[data-animate]`.
- Reduced-motion users see all content immediately, fully opaque, with no transform.
- The block may be empty or omitted entirely — do NOT add transition effects as a "reduced motion alternative."

**After (correct — no motion, no hiding):**
```css
@media (prefers-reduced-motion: reduce) {
  /* Intentionally empty — [data-animate] elements render at full opacity with no transform */
}
```

### Single AnimationController Rule (CONTEXT.md D-01)

**Requirement:** Exactly one IntersectionObserver runs per page. No duplicate observers.

**Implementation contract:**
- A single `<script is:inline>` AnimationController block lives in `BaseLayout.astro`, placed after `<Footer />` before `</body>`.
- The script must add `.js-enabled` to `<html>` before querying `[data-animate]` elements.
- The IntersectionObserver threshold stays at `0.15` (Claude's discretion per CONTEXT.md).
- `rootMargin` is not set (default `"0px"`) — adjust only if scroll trigger timing proves wrong during implementation.
- The duplicate `<script is:inline>` blocks in `FeatureCTA.astro` (lines 63-80) and `LandingCTA.astro` (lines 89-106) are removed entirely. No replacement script goes in those components.
- Pages that include both components (e.g. feature pages with both FeatureCTA and a section using LandingCTA) must trigger each `[data-animate]` element exactly once — verified by the single BaseLayout controller.

**AnimationController shape:**
```js
(function () {
  document.documentElement.classList.add('js-enabled');
  var observer = new IntersectionObserver(
    function (entries) {
      entries.forEach(function (entry) {
        if (entry.isIntersecting) {
          entry.target.classList.add('is-visible');
          observer.unobserve(entry.target);
        }
      });
    },
    { threshold: 0.15 }
  );
  document.querySelectorAll('[data-animate]').forEach(function (el) {
    observer.observe(el);
  });
})();
```

## Interaction States

No new interactive elements are introduced in this phase. The `[data-animate]` attribute and `.is-visible` class are the only state transitions in scope.

| State | Trigger | CSS Class Added |
|-------|---------|-----------------|
| Hidden (JS, motion OK) | Page load with JS + `.js-enabled` on `<html>` | `.js-enabled [data-animate]` applies `opacity:0` |
| Visible | Element enters viewport at ≥15% threshold | `.is-visible` added by observer |
| No-JS visible | Page load without JS (no `.js-enabled`) | No hiding rules apply — element naturally visible |
| Reduced-motion visible | OS `prefers-reduced-motion: reduce` | No hiding rules apply — element naturally visible |
