  <action>
The user must complete these steps in the Cloudflare dashboard (Claude cannot access the dashboard):

**Step 1: Create Cloudflare Pages project**
1. Go to Cloudflare Dashboard > Workers and Pages > Create application > Pages tab
2. Connect GitHub and select the `tuwa-website` repository
3. Configure build settings:
   - Build command: `npm run build`
   - Output directory: `dist`
4. Add build environment variable: `NODE_VERSION` = `22` (per D-10)
5. Click "Save and Deploy"
6. Wait for initial build to complete — note the assigned `*.pages.dev` subdomain

**Step 2: Add custom domain**
1. Go to the Pages project > Custom domains > Set up a domain
2. Enter `tuwa.app`
3. Cloudflare will auto-create a CNAME record (DNS is already on Cloudflare per D-11)
4. Do NOT manually create a CNAME record first — this causes 522 errors (per RESEARCH.md Pitfall 3)
5. Wait for DNS propagation (typically instant for same-account Cloudflare zones)

**Important notes:**
- Do NOT install `@astrojs/cloudflare` adapter — the site is pure static output, no adapter needed (per D-12)
- The `public/_redirects` file is already configured with legacy redirects — Cloudflare Pages processes it automatically
- `astro.config.mjs` already has `site: 'https://tuwa.app'` — no changes needed

**Caveat:** The Cloudflare dashboard UI may differ from what's described above. Confirm what you see before proceeding.

  <action>
1. Build the production output and run local Lighthouse audits:
   ```bash
   npm run build
   ```

2. Start the preview server in the background:
   ```bash
   npm run preview &
   ```

3. Run Lighthouse mobile audit (default throttling = mobile):
   ```bash
   npx lighthouse http://localhost:4321 --output=json --output-path=./lh-mobile.json --chrome-flags="--headless"
   ```

4. Run Lighthouse desktop audit:
   ```bash
   npx lighthouse http://localhost:4321 --preset=desktop --output=json --output-path=./lh-desktop.json --chrome-flags="--headless"
   ```

5. Extract Performance scores from JSON output:
   ```bash
   node -e "const r=require('./lh-mobile.json'); console.log('Mobile Performance:', Math.round(r.categories.performance.score * 100))"
   node -e "const r=require('./lh-desktop.json'); console.log('Desktop Performance:', Math.round(r.categories.performance.score * 100))"
   ```

6. **If BOTH scores >= 95:** Phase complete. Clean up lighthouse output files (`rm lh-mobile.json lh-desktop.json`).

7. **If any score < 95:** Apply fixes in priority order per D-13:
   - **First sacrifice: noise texture** — In `src/styles/global.css`, find the `body::after` rule that creates the noise texture overlay (contains `background-image: url("data:image/svg+xml` and `position: fixed`). Comment out or remove the entire `body::after` rule block. Rebuild and re-test.
   - **Second: check for lazy-loaded above-fold images** — Verify all hero-area images have `loading="eager"`.
   - **Third: check Total Blocking Time** — If TBT is high, look at the inline script in BaseLayout.astro for optimization opportunities.
   - After each fix, rebuild (`npm run build`) and re-run Lighthouse to confirm improvement.

8. Also run Lighthouse against the live deployment URL (per D-14):
   ```bash
   npx lighthouse https://tuwa.app --output=json --output-path=./lh-live-mobile.json --chrome-flags="--headless"
   npx lighthouse https://tuwa.app --preset=desktop --output=json --output-path=./lh-live-desktop.json --chrome-flags="--headless"
   ```
   Extract and verify scores >= 95.

9. Clean up all lighthouse JSON files after verification.
