   - Add a new `.hero-headline` rule OUTSIDE any media query (so it applies in all motion-preference states). Place it after the hero animation block (after the hero CSS section, roughly after the hero-related styles). The rule:
     ```css
     .hero-headline {
       font-size: clamp(32px, 7vw, 48px);
     }
     ```
     This overrides the inline `font-size: var(--text-display)` (48px fixed) on the Hero h1, providing fluid scaling from 32px at 375px to 48px at ~686px+ without adding new breakpoints. Per D-05 (no new breakpoints) and D-06 (use clamp() for fluid sizing).

     IMPORTANT: The `.hero-headline` class selector has higher specificity than the inline style ONLY if we remove the inline font-size from Hero.astro. Since inline styles beat class selectors, we must ALSO remove `font-size: var(--text-display);` from the Hero.astro h1 inline style attribute. This is handled by editing Hero.astro — remove the `font-size: var(--text-display);` line from the h1's style attribute (the clamp() rule in global.css will take over).
