## Spacing Scale

Declared values — already live as CSS custom properties in `src/styles/global.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding, hairline offsets |
| sm | 8px | Button internal padding (vertical), compact gaps |
| md | 16px | Button internal padding (horizontal), paragraph gaps |
| lg | 24px | Section sub-group spacing, card padding |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Between major content blocks within a page section |
| 3xl | 64px | Page-level section padding (top and bottom of every section) |

Exceptions:
- Mobile touch targets (nav links, FAQ toggles, CTA buttons): minimum 44px height — use `min-height: 44px` as established in MobileMenu.astro pattern.
- Feature page screenshot containers: `max-width: 320px` centered, matching Hero treatment.
- Legal prose container: `max-width: 680px` centered — narrower than feature pages for optimal line length at body size.

Source: `src/styles/global.css` `:root` spacing variables (confirmed, no changes needed)

## Typography

All sizes are CSS variables declared in `src/styles/global.css`. Use these tokens — do not hardcode pixel values.

| Role | Token | Size | Weight | Line Height | Letter Spacing |
|------|-------|------|--------|-------------|----------------|
| Display | `--text-display` | 48px | 600 (semibold) | 1.1 (`--leading-display`) | -0.02em |
| Heading | `--text-heading` | 28px | 600 (semibold) | 1.2 (`--leading-heading`) | -0.01em |
| Body | `--text-body` | 16px | 400 (regular) | 1.6 (`--leading-body`) | 0 |
| Label | `--text-label` | 13px | 400 (regular) | 1.4 (`--leading-label`) | 0.06em (all-caps contexts only) |

**Feature page heading hierarchy:**
- `<h1>`: Display (48px / 600) — feature name and outcome statement. One per page.
- `<h2>`: Heading (28px / 600) — major section titles (e.g. "How it works", "The science", "For coaches").
- `<h3>`: Body (16px / 600) — subsection labels, FAQ question text.
- Body paragraphs: Body (16px / 400 / 1.6).
- Legal prose (Privacy, Terms): Body (16px / 400 / 1.6) for paragraphs; Heading (28px / 600) for section titles.

**Long-form prose on feature and legal pages:** Use `@tailwindcss/typography` prose classes for MDX-rendered content blocks, which is already installed. Apply `prose prose-neutral` on the container; override with CSS custom properties for color so Alpino and travertine tokens remain authoritative.

Source: `src/styles/global.css` type scale + existing component patterns (confirmed in Hero.astro, Header.astro, Footer.astro)

## Color

All values are CSS custom properties declared in `src/styles/global.css`. Do not use raw hex values in components.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-bg` | #F4F1ED | Page background for all feature and legal pages |
| Secondary (30%) | `--color-surface` | #EDEAE6 | Feature page "science explainer" alternate-row backgrounds, legal page aside blocks, FAQ item backgrounds |
| Surface elevated | `--color-surface-el` | #E4E0DB | Hover states on interactive FAQ rows, hover on nav links |
| Divider | `--color-divider` | #CFCBC5 | HR elements between legal sections, FAQ item borders |
| Primary text | `--color-text-1` | #1C1915 | All headings, body text, feature page copy |
| Secondary text | `--color-text-2` | #696560 | Subheadings, supporting copy, methodology captions |
| Tertiary text | `--color-text-3` | #AFABA5 | Placeholder image labels, chart axis labels, micro annotations |
| Accent (10%) | `--color-accent` | #2B5240 | CTA buttons only (App Store CTA on each feature page, "Contact Support" on support page) |
| Accent hover | `--color-accent-hover` | #1E3D2F | CTA button hover/active state |
| Accent foreground | `--color-accent-fg` | #F4F1ED | Text on accent-background buttons |

**Accent reserved for:** App Store download CTA buttons on feature pages, the "Contact Support" action link on the support page. Not used on decorative elements, section headings, chart lines, or icon fills.

**Chart accent color:** For interactive ACWR and recovery scoring charts (D-08), use `--color-accent` (#2B5240) as the primary data series color. Use `--color-text-3` (#AFABA5) for secondary/reference lines and axis labels. Never introduce new colors in charts.

## Page Structure

### Standard Feature Pages (4 pages: Recovery Scoring, Workload Tracking, Smart Templates, Cold-Start Onboarding)

```
BaseLayout
  └── FeaturePageLayout
        ├── Hero section [data-animate]
        │     ├── <h1> — outcome statement (display / 600)
        │     ├── <p> — one-sentence hook (body / 400 / color-text-2)
        │     └── App Store badge (mobile) / badge + QR (desktop)
        ├── Screenshot section [data-animate]
        │     └── ScreenshotBlock — centered, max-width 320px, border-radius var(--radius-lg), box-shadow 0 4px 24px rgba(0,0,0,0.08)
        ├── Explanation section [data-animate]
        │     ├── <h2> "How it works" (heading / 600)
        │     └── Body paragraphs — 800-1200 words total per page
        ├── Science / Methodology section [data-animate] (surface bg: --color-surface)
        │     ├── <h2> "The science behind it" (heading / 600)
        │     ├── Body paragraphs
        │     └── RecoveryChart or AcwrChart (where applicable, as Astro island)
        └── FeatureCTA section [data-animate]
              └── "Download on the App Store" button
```

### Coaching Page (1 page — extended layout)

Same structure as standard feature pages plus:
- Additional `<h2>` sections: "For athletes", "For coaches", "Team workflows", "Invite your athletes"
- Each perspective section is visually separated with `--color-divider` horizontal rule
- Invite flow block shows code/email/NFC mechanics in a surface-bg card

### Legal Pages (Privacy Policy, Terms of Service)

```
BaseLayout
  └── LegalPageLayout
        ├── Page header
        │     ├── <h1> page title (heading / 600) — NOT display size
        │     └── "Last updated: [date]" (label / 400 / color-text-3)
        └── Prose container (max-width: 680px, margin: 0 auto)
              └── MDX content with @tailwindcss/typography prose styles
                    ├── <h2> section headers (heading / 600)
                    ├── <h3> sub-headers (body / 600)
                    ├── <p> paragraphs (body / 400 / 1.6)
                    └── <hr> section dividers (color-divider)
```

### Support Page

```
BaseLayout
  └── LegalPageLayout

        ├── FaqAccordion (5-8 questions)

              ├── Contact email as mailto link (body / accent color)
```

## Screenshot / Placeholder Treatment

Decision D-02 and D-04 from 03-CONTEXT.md:

**When screenshot available:** Bare screenshot (no device mockup), `border-radius: var(--radius-lg)` (8px), `box-shadow: 0 4px 24px rgba(28, 25, 21, 0.10)`. Max-width: 320px centered. Use `<Image>` component with `format="webp"` and `quality={90}`.

**When screenshot not yet available (placeholder):** Render a `--color-surface` rectangle at the same max-width 320px and aspect ratio 9:19.5 (portrait phone proportions). Centered label in `--color-text-3` at 13px: "[Feature Name] Screenshot — coming soon". No border, no shadow on placeholder.

This treatment is distinct from the Hero section's full-bleed device mockup (Dashboard_framed.png) — feature page screenshots are inline content, not hero focal points.

## Accessibility Contract

- Every `<img>` and `<Image>` must have descriptive `alt` text (not empty, not "screenshot"). Alt text describes what is shown in the UI and its relevance to the feature.
- Each page has exactly one `<h1>`. Heading hierarchy is `h1 → h2 → h3` with no skipped levels.
- Interactive FAQ (FaqAccordion): use `<details>/<summary>` — native keyboard accessible, no ARIA required.
- All App Store badge links: `aria-label="Download Tuwa on the App Store"`.
- `prefers-reduced-motion` respected via existing CSS in global.css — no new motion CSS should bypass this.
- Focus ring: `2px solid var(--color-accent)` with `outline-offset: 2px` — already declared in global.css `:focus-visible`.
- Chart islands: include `aria-label` on the `<canvas>` element describing what data is shown (e.g. "Line chart showing 28-day ACWR trend").
