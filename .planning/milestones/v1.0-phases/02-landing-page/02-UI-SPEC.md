Accent reserved for:

3. "Get the App" header CTA (already built in Phase 1)

Accent is NOT used for: feature card hover states, section dividers, icon fills, decorative elements.

Feature card hover state: background shifts from `--color-surface` (#EDEAE6) to `--color-surface-el` (#E4E0DB). No accent. Border: `--color-divider` (#CFCBC5) at 1px.

## Component Inventory

### Hero Section (`src/components/Hero.astro`)

Layout: centered column, max-width 768px, horizontally centered in page.

```
[headline — display 48px semibold]
[subtitle — body 16px regular, --color-text-2, max-width 560px]
[App Store badge — accent button, 44px tall minimum]
[device mockup — iPhone frame, screenshot inside]
```

Vertical rhythm (top to bottom):
- Section padding-top: `--space-3xl` (64px)
- Headline → subtitle gap: `--space-lg` (24px)
- Subtitle → badge gap: `--space-xl` (32px)
- Badge → mockup gap: `--space-2xl` (48px)
- Section padding-bottom: `--space-3xl` (64px)

Device mockup treatment (D-02 Dieter Rams aesthetic):
- iPhone outline: thin 1.5px stroke in `--color-divider` (#CFCBC5)
- Border radius on frame: 44px (approximates iPhone 16 corner radius)
- No drop shadow, no gradient, no reflection, no ornamental bezel
- Screenshot fills the inner screen area with 4px inset from frame
- Side buttons: omit — honest presentation, not a replica
- Max width of mockup container: 320px on desktop, 280px on mobile (scales down)
- Background behind mockup: none — travertine page background shows through

Desktop QR code (D-07 — hidden on mobile via CSS `md:flex` / `flex md:hidden`):
- QR code: 120x120px, `--color-text-1` modules on transparent background
- Accompanies App Store badge horizontally on desktop (badge left, QR right)
- QR label: "Scan to download" in label size (13px), `--color-text-2`

### Feature Grid (`src/components/FeatureGrid.astro`)

5 cards, grid arrangement: 3-column on desktop (`md:grid-cols-3`), 2-column on tablet (`sm:grid-cols-2`), 1-column on mobile.

Fifth card centers in the last row on desktop (spans middle column via CSS or explicit offset).

Card anatomy (D-04 — minimal, teaser only):
```
[icon — 24x24 SVG, --color-accent fill, top-left of card]
[category label — 13px, --color-text-3, ALL CAPS, 0.06em tracking]
[title — body 16px semibold (600), --color-text-1]
[one-line description — body 16px regular, --color-text-2]
[→ arrow or implicit click target — entire card is the link]
```

Card styles:
- Background: `--color-surface` (#EDEAE6)
- Border: 1px solid `--color-divider` (#CFCBC5)
- Border radius: `--radius-lg` (8px)
- Padding: `--space-md` (16px)
- Hover: background transitions to `--color-surface-el` (#E4E0DB), transition 150ms ease
- The entire card `<a>` is the interactive element — no separate "Learn more" link

Cards (in order, D-05):
1. Recovery Scoring — links to `/features/recovery-scoring`
2. Workload Tracking — links to `/features/workload-tracking`
3. Smart Templates — links to `/features/smart-templates`
4. Cold-Start Onboarding — links to `/features/cold-start`
5. Coaching — links to `/features/coaching`

Source: CONTEXT.md D-04, D-05

### Final CTA Section (`src/components/LandingCTA.astro`)

Background: `--color-surface` (#EDEAE6) — creates visual break from travertine page bg.
Layout: centered column, max-width 640px.

```
[closing headline — heading 28px semibold]
[supporting copy — body 16px regular, --color-text-2, 1–2 sentences]
[App Store badge + QR code (desktop) / badge only (mobile)]
```

Same badge + QR layout as hero (D-08 cross-references D-07).

Section padding: `--space-3xl` (64px) top and bottom.

### Scroll Animations (D-06)

Sections that receive fade-in on scroll:
1. Feature grid section (entire section fades in together)
2. Final CTA section

Hero section does NOT animate — it is above the fold, visible on load.

Animation contract:
- `opacity: 0` → `opacity: 1`
- `transform: translateY(16px)` → `transform: translateY(0)`
- Duration: 400ms
- Easing: `ease-out`
- Trigger: `IntersectionObserver` with `threshold: 0.15`
- No stagger between individual feature cards — grid animates as a unit
- Implementation: CSS `@keyframes` + `data-animate` attribute + inline `<script is:inline>` (matches MobileMenu pattern from Phase 1)
- `prefers-reduced-motion`: when active, skip transform, keep instant opacity change only (respect accessibility)

## Interaction States

All interactive elements must have explicit styles for:

| State | Treatment |
|-------|-----------|
| Default | As specified per component above |
| Hover | 150ms ease color/background transition |
| Focus-visible | 2px solid `--color-accent` (#2B5240), 2px offset (already in global.css) |
| Active/pressed | Background one step darker (use `--color-accent-hover` for accent elements, `--color-surface-el` for cards) |
| Disabled | Not applicable — no disabled states on landing page |

## Accessibility Contract

- Hero `<h1>`: the hero headline. One `<h1>` per page.
- Section headings: `<h2>` for "Everything you need to train without guessing" and the final CTA headline.
- Feature card titles: `<h3>` within the section.
- Device mockup `<img>` alt: "Tuwa app showing today's recovery score of 82 — HRV in green zone, sleep 7.5 hours."
- App Store badge `<img>` alt: "Download on the App Store"
- QR code `<img>` alt: "QR code to download Tuwa from the App Store"
- All section elements use semantic HTML: `<section>`, `<article>` for cards, `<nav>` where applicable.
- `prefers-reduced-motion` media query wraps all animation keyframe rules.
